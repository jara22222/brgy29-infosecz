# Project Structure & Guidelines

This project is a modern web application built with **Laravel 11**, **React 19**, and **Inertia.js**.

## Core Stack
- **Backend:** Laravel 11, PHP 8.2+
- **Frontend:** React 19, Inertia.js
- **Build Tool:** Vite, TypeScript
- **Routing:** `@laravel/vite-plugin-wayfinder` for automatic route generation.
- **Styling:** TailwindCSS 4, Radix UI Primitives, class-variance-authority, clsx, tailwind-merge.
- **Animations:** Framer Motion, tw-animate-css.

## Directory Structure

### Backend (Laravel)
- `/app`: Contains core business logic, Models, Controllers.
- `/routes`: Web and API routes (Note: Wayfinder is enabled for frontend routing).
- `/database`: Migrations, Factories, and Seeders.
- `/config`: Configuration files for Laravel packages.
- `/storage`: App storage (logs, uploads, cache).

### Frontend (React/Inertia) - `/resources`
- `/resources/js/app.tsx`: Main React/Inertia entry point.
- `/resources/js/ssr.tsx`: Server-Side Rendering entry point.
- `/resources/js/components`: Reusable UI components (often Radix UI based).
- `/resources/js/layouts`: Page layouts for Inertia pages.
- `/resources/js/pages`: Inertia.js page components.
- `/resources/js/contexts`: React context providers.
- `/resources/js/hooks`: Custom React hooks.
- `/resources/js/lib`: Utility functions (e.g., `utils.ts` for cn() with clsx/tailwind-merge).
- `/resources/js/types`: TypeScript type definitions.
- `/resources/js/actions`: Frontend actions/API calls.
- `/resources/js/routes`: Generated by Wayfinder.
- `/resources/css/app.css`: Main Tailwind CSS entry file.

## Development Commands
- `npm run dev`: Starts Vite dev server.
- `php artisan serve`: Starts Laravel dev server.
- `npm run dev:all`: Starts both using concurrently.
- `npm run build`: Builds assets for production.
- `npm run lint`: Runs ESLint for formatting.

## Best Practices
1. **Components:** Use Radix UI primitives with Tailwind for accessible, unstyled components.
2. **Types:** Strictly type React components and Inertia props with TypeScript.
3. **Styling:** Use Tailwind CSS 4 utility classes. Merge classes using `clsx` and `tailwind-merge`.

# Design Elements

This project employs a modern, accessible, and highly animated design system based on React and TailwindCSS.

## 1. CSS Framework
- **Tailwind CSS 4 (`tailwindcss`):** The core utility-first CSS framework for styling all components.
- **Tailwind Plugins:** Uses standard Tailwind 4 configuration with Vite integration (`@tailwindcss/vite`).

## 2. Component Library & Accessibility
- **Radix UI Primitives (`@radix-ui/react-*`):** Unstyled, accessible UI components used as the foundation for the design system.
  - Avatars, Checkboxes, Collapsibles, Dialogs, Dropdowns, Labels, Navigation Menus, Popovers, Radio Groups, Selects, Separators, Slots, Toggles, Tooltips.
- **Headless UI (`@headlessui/react`):** Used in conjunction with Radix for specific headless component needs.

## 3. UI Architecture
- **Class Merging:** Uses `clsx` and `tailwind-merge` for conditional and conflict-free Tailwind class combination.
- **Variant Management:** Uses `class-variance-authority` (cva) to define component variants (e.g., primary vs secondary buttons, sizes).

## 4. Animation & Motion
- **Framer Motion (`framer-motion`):** For complex, physics-based animations, layout transitions, and interactive micro-animations.
- **Tailwind Animate (`tw-animate-css`):** For utility-class based CSS animations.

## 5. Icons & Typography
- **Icons:** Uses `lucide-react` for a consistent, clean, and customizable SVG icon set.

## 6. Complex Data & Interactions
- **Tables:** Uses `@tanstack/react-table` for highly customizable, headless data tables.
- **Dates:** Uses `date-fns` for manipulation and `react-day-picker` for date picking interfaces.
- **Forms/OTP:** Uses `input-otp` for one-time password inputs.
- **Notifications:** Uses `react-hot-toast` for elegant, animated toast notifications.

## 7. Design Philosophy
- **Unstyled Foundation:** The UI relies on headless components (Radix) styled entirely with Tailwind, providing total control over the design system without overriding default styles.
- **Micro-interactions:** Enhanced with Framer Motion to provide a fluid and responsive user experience.
